ColorForge User Documentation

1. Overview

ColorForge is a professional color engineering tool designed for designers and anyone working with color. It helps you pick colors from images, analyze accessibility using WCAG standards, generate usable color palettes, and export color codes for design and development workflows.

ColorForge is privacy-first and runs entirely on your device. No images, colors, or data are uploaded or tracked.

1.1 Who This Is For

  • UI / UX designers
  • Web and product designers
  • Brand designers
  • Developers working with design handoff
  • Anyone who needs reliable color and accessibility decisions

2. Quick Start

Get value from ColorForge in under two minutes.

2.1 Pick a Color

  • Open the Color Room
  • Use the Color Wheel or Eyedropper to select a color from an image

2.2 Check Accessibility

  • View the WCAG contrast rating (AA, AAA, or Fail)
  • Results update instantly as you adjust the color

2.3 Generate Palettes

  • Browse automatically generated color harmonies
  • Click any palette color to make it your active color

2.4 Export Color Codes

  • Copy HEX, RGB, HSL, or advanced formats
  • Use them directly in design tools or code

3. Picking Colors

3.1 Using the Color Wheel

The Color Wheel is the primary way to select colors.

  • Drag inside the wheel to change hue and saturation
  • Adjust brightness to control lightness
  • All changes are reflected instantly

3.2 Using the Eyedropper

The Eyedropper allows you to sample colors directly from images.

  • Load or paste an image into the Color Room
  • Click anywhere on the image to sample a color
  • The sampled color becomes the active color

4. Accessibility & WCAG Ratings

4.1 What ColorForge Checks

ColorForge evaluates color contrast using WCAG 2.x guidelines against light and dark text where applicable.

4.2 WCAG Rating Levels

  • AAA – Excellent accessibility
    • Recommended for body text and critical information
  • AA – Acceptable accessibility
    • Suitable for most interface elements
  • Fail – Does not meet accessibility standards
    • Should not be used for readable text

4.3 Why Accessibility Matters

Accessible colors improve readability, usability, and inclusivity. In many regions, WCAG compliance is also a legal requirement for public-facing products.

5. Palette Generation

5.1 How Palettes Work

ColorForge generates palettes based on established color harmony principles.

5.2 Available Palette Types

  • Complementary
  • Analogous
  • Triadic
  • Split Complementary (if available)

5.3 Using Palettes Effectively

  • Palettes update automatically when the main color changes
  • Click a palette swatch to set it as the active color
  • Use palettes to build consistent color systems

6. Color Formats & Exporting

6.1 Supported Color Formats

  • HEX
  • RGB
  • HSL
  • CMYK
  • LAB / LCH (Advanced)

6.2 Exporting Colors

  • Click any color value to copy it to your clipboard
  • Enable "Show Advanced" to access additional formats

6.3 Common Usage Scenarios

  • Web design: HEX or RGB
  • Design tools: HEX
  • Developer handoff: Multiple formats

7. Best Practices

7.1 Designing Accessible Interfaces

  • Target AA minimum, AAA for text-heavy content
  • Test accessibility early in the design process
  • Do not rely on color alone to convey meaning

7.2 Building Color Systems

  • Start with a strong base color
  • Use generated palettes for consistency
  • Validate contrast across all combinations

8. Troubleshooting & FAQ

8.1 Why does my color fail WCAG?

The contrast between foreground and background is too low. Adjust brightness or select an alternative palette color.

8.2 Are these results legally compliant?

ColorForge follows WCAG guidelines, but final compliance depends on your full design context.

8.3 Does ColorForge collect my data?

No. All processing happens locally on your device.

9. Versioning
